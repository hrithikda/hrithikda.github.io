LLM Observability Platform

Overview

A production-grade evaluation and observability platform for LLM applications and RAG pipelines. Addresses the core problem of shipping LLM systems that look good in demos but degrade silently in production — through automated scoring, drift detection, and regression gating.

What was built

  • RAGAS evaluation suite: Automated scoring across faithfulness, answer relevancy, context precision, and context recall — run as a CI gate on every pipeline change.
  • LLM-as-judge layer: Secondary evaluation using a critique LLM to score response quality, factual grounding, and instruction-following on a per-question basis.
  • Golden test set management: Versioned dataset of question/context/ground-truth triples with tooling to expand, curate, and diff across dataset versions.
  • Drift detection: Score distribution monitoring over time — alerts when mean faithfulness or relevancy drops below threshold across a rolling evaluation window.
  • MLflow integration: All evaluation runs logged as experiments with metric history, dataset version, model version, and prompt template tracked for full reproducibility.
  • FastAPI + dashboard: REST API for triggering evaluations and querying results. Streamlit dashboard showing score trends, per-question breakdowns, and regression alerts.

Why it matters

Inspired directly by the RAGAS evaluation suite built for the Municipal Sustainability AI project (Graduate Research Assistant, UIC) where 12 automated checkpoints across 7 prototype generations raised retrieval precision by 42%. This platform generalizes that pattern into a reusable service — the infrastructure layer that makes LLM systems trustworthy enough to deploy and maintain.

Project Info

  • Category: LLM Evaluation / MLOps
  • Metrics tracked: Faithfulness, relevancy, context precision/recall, LLM-judge score
  • Key feature: CI-gated regression testing on every release
