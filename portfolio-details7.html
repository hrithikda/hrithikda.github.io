Municipal Sustainability AI

Overview

Built as an independent study (IDS 596) at UIC, this project traces the full arc of modern AI system architecture — from a single LLM extracting structured text to a multi-agent system with specialist roles, self-verification, and workflow orchestration. The domain is municipal climate and sustainability policy: scraping, indexing, and reasoning over public data from 15 US states.

What was built

  • Web scraper targeting 41 climate and sustainability data sources — producing a final corpus of 35 documents, 1,895,253 characters.
  • DSPy 2.6.27 pipeline with Ollama (llama3.1:8b) for structured information extraction from unstructured municipal policy text.
  • ChromaDB vector store with nomic-embed-text (768-dim) embeddings — 714 document chunks indexed for semantic search with retrieval latency under 150ms.
  • Multi-agent routing: specialist LLM roles for different query types, OutputCritic pattern for self-verification, and force-finish mechanisms for ReAct agent reliability.
  • scikit-learn scoring model trained on 35 labeled samples — 63% MAE improvement (13.51 → 5.01) over baseline.
  • n8n workflow orchestration for pipeline automation; FastAPI backend; Streamlit dashboard for cross-city comparative queries.
  • Docker Compose deployment; 12/12 system checkpoints passing; full pipeline completes in under 3 minutes.

Why it matters

The project demonstrates architectural progression — not just a RAG chatbot, but a system that shows when and why you'd add each layer: structured extraction with DSPy over naive prompting, semantic chunking over keyword search, agent routing over single-model calls, and workflow orchestration over ad-hoc scripts. Each step is motivated and measurable.
