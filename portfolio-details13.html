AutoDS — Autonomous Data Science Agent

Overview

A multi-agent system that automates the full data science workflow given only a CSV file and a target column. Specialist agents handle EDA, feature engineering, model selection, hyperparameter tuning, and report generation — orchestrated by LangGraph with CrewAI agent roles.

What was built

  • Agent roles: EDA Agent (profiling, missing values, distributions, correlations), Feature Engineer (encoding, scaling, interaction terms, leakage detection), Model Selector (baseline benchmarking across 6 algorithms), Tuner (Optuna-based HPO), and Reporter (narrative report generation).
  • LangGraph orchestration: Stateful workflow DAG with conditional routing — model selection outcome determines whether to proceed to tuning or loop back to feature engineering.
  • Tool use: Each agent has access to code execution tools, allowing them to run Python, inspect outputs, and revise their approach based on actual results rather than just planning.
  • Model selection: Benchmarks Logistic Regression, Random Forest, XGBoost, LightGBM, CatBoost, and SVM across cross-validated AUC/RMSE. Recommends top-2 for tuning.
  • Optuna tuning: Automated hyperparameter search with pruning — 100 trials, early stopping, and study visualization.
  • Report output: Auto-generated markdown/PDF report covering data quality findings, feature importance, model comparison, tuning results, and deployment recommendations.
  • Streamlit interface: Upload dataset, set target column and task type, watch agents execute in real time, download final report and best model artifact.

Why it matters

AutoML tools like AutoSklearn automate the search but black-box the reasoning. AutoDS exposes the decision-making — you can see why the EDA agent flagged leakage, why the model selector picked XGBoost over LightGBM, and what the tuner changed. The agentic approach makes automation explainable, not just fast.
