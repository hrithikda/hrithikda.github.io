Intelligent Operations Command Center

Overview

An end-to-end operational intelligence platform that simulates enterprise-scale observability data, detects anomalies using an ensemble ML approach, forecasts capacity issues, and generates RAG-powered root cause analyses through LLM reasoning — all orchestrated, tracked, and containerized for production deployment.

What was built

  • Five-layer architecture: ingestion → feature engineering → anomaly detection → forecasting → LLM reasoning, with each layer independently testable.
  • Ingestion: configurable data generators simulating metrics, logs, tickets, and transactions — 17,280+ events processed per run. Apache Airflow DAGs for pipeline orchestration.
  • Feature engineering: 133 features including temporal encoding, rolling statistics, and lag features across all data sources.
  • Anomaly detection: ensemble of Isolation Forest (scikit-learn) and PyTorch autoencoder with learned combination weights. Precision 94.4%, Recall 77.1%, F1 84.9%, False Positive Rate 0.15%.
  • Forecasting: LightGBM regressor for 1-minute CPU load prediction — 14.84% MAPE, a 35.7% improvement over baseline. Tracked with MLflow.
  • LLM reasoning: RAG system using ChromaDB (Sentence Transformers) for historical incident retrieval, fed into Anthropic Claude API for structured root cause analysis.
  • Serving: FastAPI with six endpoints covering health, anomalies, metrics, forecasts, model performance, and LLM analysis. Streamlit command center dashboard.
  • Infrastructure: Docker Compose (dev) and PostgreSQL + TimescaleDB (prod), GitHub Actions CI/CD, unit and integration test suite.

Why it matters

Most anomaly detection projects stop at a model evaluation notebook. This one is structured as a deployable service: data flows through Airflow, models are versioned in MLflow, predictions are served over a REST API, and explanations are generated on-demand by an LLM with access to historical incident context. The ensemble approach is deliberate — Isolation Forest catches global outliers while the autoencoder learns normal operational patterns from reconstructed sequences, and their combination reduces false positives to 0.15%.
