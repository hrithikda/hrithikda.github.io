SQL Co-Pilot

Overview

A natural language to SQL system that translates plain-English questions into executable queries against a user-provided database schema. Goes beyond basic NL-to-SQL by handling multi-table joins, window functions, aggregations, and nested subqueries — with query validation before execution.

What was built

  • Schema-aware prompting: Dynamic schema injection into each prompt — table names, column types, foreign key relationships, and sample values. Keeps the LLM grounded in the actual database structure.
  • Few-shot examples: Per-schema few-shot example bank automatically selected by semantic similarity to the input question, improving accuracy on complex query patterns.
  • Query validation: Generated SQL is parsed and validated against the schema before execution — catching hallucinated column names or incorrect table references before they hit the database.
  • Self-correction loop: On execution error, the error message is fed back to the LLM with the failed query for one round of self-correction.
  • FastAPI backend: Endpoints for schema registration, question submission, and result retrieval. Supports PostgreSQL and Snowflake dialects.
  • Streamlit interface: Upload a schema, ask questions in plain English, see the generated SQL alongside results — with query history and export.

Why it matters

Text-to-SQL is notoriously brittle on complex schemas — models hallucinate columns, miss joins, or generate syntactically valid but semantically wrong queries. The schema injection + validation + self-correction loop addresses each failure mode directly, making the system reliable enough for non-technical users to query production databases without writing a line of SQL.
